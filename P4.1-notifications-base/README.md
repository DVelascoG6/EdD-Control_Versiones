# Práctica: Refactorización de NotificationManager

## Objetivo
Refactorizar un sistema de notificaciones monolítico en un diseño modular y mantenible.

## Estructura del Proyecto
```bash
/P4.1-notifications-base
├── README.md # Este archivo
├── .gitignore # Archivos a ignorar (localizado en un directorio superior)
├── NotificationManager.java
├── Main.java
├── P4.1-GitHub_Master-Colaboración_Controlada.pdf
└── /documentacion # Capturas y evidencias
```

## Cómo Empezar
1. **Fork** este repositorio (botón arriba a la derecha)
2. **Clone** en tu máquina local:
   ```bash
   git clone https://github.com/TU_USUARIO/proyecto-notificaciones.git
   cd P4.1-notifications-base
   ```
3. Crea una rama para tu trabajo:
    ```bash
    git checkout -b feature/refactor-notifications
    ```

## Sugerencias
- Commits atómicos (un cambio por commit)
- Mensajes descriptivos (ver convención abajo)
- Issues creados en GitHub
- Pull Request bien documentado
- Code Review a compañero

## Convención de Commits
feat:     Nueva funcionalidad
fix:      Corrección de bug
refactor: Cambio que no añade funcionalidad ni corrige bug
docs:     Documentación
test:     Pruebas
style:    Formato, punto y coma, etc.
